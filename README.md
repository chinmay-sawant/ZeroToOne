# Zero to One

Language roadmaps for **college graduates** — from first program to production-ready systems.

Each track uses one continuous domain: an **Employee Management** application. You start with a simple in-memory CLI and grow the same product into an expert-level system.

This repo also ships a **minimal React checklist platform** under `frontend/` that:

- Renders each language’s prompt/readme
- Lets you tick beginner → expert checklist items
- Saves progress in **browser localStorage** (per device/browser)
- Loads content from **local files** in dev, and from **GitHub `master`** when deployed (GitHub Pages)

## Checklist app (`frontend/`)

### Stack

- React + TypeScript (Vite)
- Tailwind CSS + **shadcn/ui** (black & white, no gradients)
- `react-markdown` for readme rendering
- Progress: `localStorage` keys `zerotoone:checklist:v1`, `zerotoone:active-language`

### Content loading

| Environment | How readmes are loaded |
|-------------|------------------------|
| **Local** (`localhost` / `127.0.0.1`) | `frontend/public/content/{java,python,golang}.md` (copied from language folders) |
| **GitHub Pages** | `https://raw.githubusercontent.com/chinmay-sawant/ZeroToOne/master/{lang}/readme/{lang}.md` |

Production always reflects the **master** branch prompts without baking markdown into the JS bundle.

### Makefile (recommended)

From the **repository root**:

```bash
make install   # npm install in frontend/
make dev       # sync content + start Vite
make build     # delete docs/ if present, sync content, build → docs/
make preview   # preview production build (base /ZeroToOne/)
make lint      # oxlint
make content   # only sync language readmes → frontend/public/content
make clean     # remove docs/ + generated content
make help      # list targets
```

### npm (from `frontend/`)

```bash
cd frontend
npm install
npm run dev      # sync content → public/content, start Vite
npm run build    # delete ../docs/ if present, sync content, build into ../docs/
npm run preview  # preview the docs build
npm run lint
```

`npm run build` / `make build` output:

- Cleans existing **`docs/`** at the **repository root**
- Regenerates **`docs/`** for GitHub Pages (`base: /ZeroToOne/`)

### GitHub Pages

1. Push `master` with the built `docs/` folder (or build in CI and commit/deploy `docs`).
2. Repo **Settings → Pages →** source: Deploy from branch **`master`** / folder **`/docs`**.
3. Site URL shape: `https://chinmay-sawant.github.io/ZeroToOne/`

Remote:

```bash
git remote add origin https://github.com/chinmay-sawant/ZeroToOne.git
```
